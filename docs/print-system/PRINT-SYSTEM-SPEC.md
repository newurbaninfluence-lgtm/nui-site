## CLIENT UX FLOW

### Door 1: Print Only (Client Has Artwork)
1. Client lands on /print or gets texted a link
2. Sees product grid with photos and starting prices
3. Taps product → size/qty selector → price updates live
4. "I have my artwork" → file upload (PDF, AI, PNG, JPG)
5. Add to cart → continue shopping or checkout
6. Checkout: name, email, phone, shipping address
7. No account required — just submit
8. Order hits Supabase print_requests
9. Faren gets email + SMS notification
10. Client gets confirmation email with order summary
11. Faren downloads files, submits to Signs365/Knello
12. Marks status updates in admin → client gets notified

### Door 2: Design + Print (Client Needs Design)
1. Same product browsing experience
2. "I need design help" → shows design fee added to price
3. Client selects products + design add-on
4. Checkout includes: "Describe what you need" text field
5. Order hits Supabase with design_needed: true flag
6. Shows up in Jobs Board as new design job
7. Faren creates design via normal moodboard → brand guide flow
8. Client approves design
9. Faren submits approved files to printer
10. Same fulfillment flow from there

### Door 3: Branding Client Print Upsell (Existing Flow)
1. Client completes branding project (brand guide delivered)
2. Admin marks job "done" → triggers print upsell email
3. Email shows: "Your brand is ready — now let's print it"
4. Links to /print with their brand assets pre-loaded
5. Client orders print products with their new designs
6. No design fee — artwork already exists from branding project

## TECHNICAL ARCHITECTURE (Performance-First)

### What Loads on /print Page
1. print-store.js (~15-20KB) — product catalog, calculator, cart, UI
   - Pricing data embedded in JS (no API call)
   - Cart state in localStorage
   - Quote calculation is pure math, no server calls
   - Product images lazy-load (same pattern as portfolio)

2. Product photos — compressed JPGs, lazy-loaded on scroll
   - Use Signs365 product photos or shoot your own
   - Thumbnails ~50KB each, only load visible ones

### What Does NOT Load
- No impact on homepage, portfolio, services, or any other page
- No external pricing APIs
- No database reads until order submission
- No heavy frameworks — vanilla JS like everything else

### Supabase Tables
- print_requests (existing) — order storage
  Add columns: items (JSONB array), design_needed (bool), 
  artwork_urls (text array), shipping_address (JSONB),
  vendor_routing (text), total_price (numeric), 
  wholesale_cost (numeric, admin-only), profit (numeric, admin-only)

### Netlify Function
- submit-print-order.js — validates order, writes to Supabase,
  sends confirmation email to client, notification to Faren via email+SMS

### Shipping Logic
- Signs365 items: flat $10 added (passed through at cost)
- Knello items: calculated by zone or flat metro rate
- Mixed orders (both vendors): separate shipping lines
- Free shipping threshold? (optional — drives larger orders)

### Instant Quote Calculator Logic
```
clientPrice = wholesale × markup (2.5x baseline, never below 2x)
bundlePrice = sum(items) × bundleDiscount (bundles priced as packages, not itemized)
designFee = per product type (see design pricing table above)
shipping = vendor-specific (Signs365 $10 flat, Knello by distance)
total = sum(clientPrices) + designFees + shipping
```

### Admin View (Jobs Board Integration)
Print orders appear in existing Jobs Board with:
- Product list with quantities
- Client price vs wholesale cost (margin visible to admin only)
- Artwork status: ✅ Uploaded / ⏳ Needs Design / ❌ Missing
- Vendor routing badge: "Signs365" or "Knello"
- Status flow: New → Files Received → At Printer → Shipped → Delivered
- One-click download all artwork files
- Quick-link to open Signs365/Knello order page
